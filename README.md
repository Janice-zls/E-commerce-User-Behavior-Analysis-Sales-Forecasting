# E-commerce-User-Behavior-Analysis-Sales-Forecasting
## 一、项目简介
本项目完整实现电商全链路数据分析与运营决策建模，围绕四大核心研究问题依次完成数据预处理、用户行为聚类、个性化推荐、销量时序预测全流程建模。基于电商用户浏览、加购、收藏、下单、商品、促销多源数据集，融合无监督聚类算法与时间序列预测模型，挖掘用户分层特征、商品销售周期规律，构建多策略推荐系统并输出可落地的库存、精准营销优化方案，形成一套闭环式电商智能运营分析框架，可直接为电商精细化运营提供数据支撑与量化决策依据。

本项目基于电商五源异构数据集，搭建从数据清洗、用户分层、智能推荐到销量预测的全链路运营决策体系。首先完成缺失 / 异常数据修正、多表融合，通过时序分析挖掘月度、季度销售周期，结合卡方、相关检验量化促销、地域、年龄等核心影响因子；构建多维度用户行为指标，采用 K-Means 完成用户聚类，结合 RFM 模型划分核心、沉睡五类用户并搭建多维画像；对比协同过滤、内容推荐、混合推荐算法，验证混合模型均衡精准度与多样性；搭建 ARIMA 与指数平滑双时序预测模型，优选 ARIMA 实现商品销量预测，配套 EOQ 库存规则与分层差异化营销策略。实证显示差异化优惠可提升销量 12.5%、转化率 8.2%，整套模型可直接落地电商精细化运营，具备较强行业复用价值。

## 三、四大核心模块功能说明
### 模块1：数据清洗与销售规律分析
1. 处理多表异构数据，完成缺失值填充、异常订单过滤、重复数据去重；
2. 时间维度拆分，按月/季度绘制销售时序曲线，识别销售淡旺季周期；
3. 相关性、显著性检验，量化折扣、地域、人群对销量的影响程度；
4. 输出整体电商市场基础运营规律。

### 模块2：用户行为指标构建与聚类分群
1. 基于用户全行为链路搭建RFM扩展指标：活跃度、复购、转化率、停留时长等；
2. 使用轮廓系数确定最优聚类数量，采用K-Means实现用户自动分层；
3. 划分高价值核心用户、普通活跃用户、沉睡流失用户等群体，解析各群体行为差异。

### 模块3：用户画像构建与多策略推荐模型
1. 基于聚类结果为每类用户生成年龄、偏好品类、消费能力完整用户画像；
2. 搭建三套推荐算法并横向对比：协同过滤、内容特征推荐、混合加权推荐；
3. 使用精确率、覆盖率、多样性多维度评估推荐效果，选定最优混合推荐策略；
4. 针对不同用户群体设计差异化商品推送方案。

### 模块4：销量时间序列预测与运营策略优化
1. 搭建ARIMA、Holt-Winters指数平滑两种时序预测模型，对比预测精度；
2. 对各品类商品未来周期销量进行预测，计算预测误差MAPE；
3. 结合预测结果优化库存补货阈值，避免缺货与库存积压；
4. 针对分层用户设计差异化促销优惠，量化测算销量、转化率提升幅度。

## 四、技术栈与使用算法
### 编程语言&工具
Python 3.8+ | Pandas | NumPy | Matplotlib | Seaborn | Scikit-learn | Statsmodels
### 核心算法
1. 数据处理：缺失值填充、箱型图异常检测、多表关联融合
2. 用户分层：RFM特征工程、K-Means聚类、轮廓系数聚类评价
3. 推荐系统：协同过滤、余弦相似度、混合加权推荐
4. 时序预测：ARIMA时序模型、Holt-Winters指数平滑
5. 统计分析：皮尔逊相关系数、描述性统计、误差指标MAPE

## 五、运行环境与使用步骤
1. 环境配置：安装对应Python第三方库，`pip install pandas numpy scikit-learn statsmodels matplotlib seaborn`
2. 数据准备：将全部原始数据集放入【数据】文件夹，修改代码内文件读取路径
3. 分步执行：按问题1→问题2→问题3→问题4顺序运行代码，保证前序清洗、特征数据生成完整
4. 结果输出：每个问题文件夹自动生成可视化图表、分析报表、模型评估结果
5. 成果查阅：完整建模逻辑、结论可查阅`作品.docx`与项目PDF文档

## 六、项目核心创新与价值
1. 全链路闭环建模：从数据清洗、用户分群、个性化推荐到销量预测一体化，贴合电商真实运营流程；
2. 算法融合创新：将聚类用户分层结果融入推荐与时序预测，实现“千人千面”精准运营；
3. 可落地量化决策：所有模型输出配套可直接使用的库存、营销优化策略，并量化收益提升效果；
4. 高复用性：框架适配绝大多数线上零售电商，更换数据集即可快速复用于不同行业店铺分析。

## 七、模型局限性与优化方向
### 现有局限
1. 未引入外部突发变量（节假日大促、竞品活动、物流波动）；
2. 新用户冷启动场景下推荐模型效果存在衰减；
3. 仅使用传统时序模型，未接入LSTM等深度学习时序算法。
### 后续优化方向
1. 引入外部特征，构建多变量时序预测模型；
2. 加入冷启动解决方案，新增基于商品热度的基础推荐；
3. 尝试LSTM、Prophet等深度学习时序模型进一步提升预测精度；
4. 开发简易可视化交互页面，实现运营数据实时查询。

线上综合电商、零售店铺、直播电商平台的用户精细化运营、库存管理、精准营销、销量预估业务场景。
